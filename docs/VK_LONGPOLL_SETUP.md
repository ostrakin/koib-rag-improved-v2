# VK Long Poll бот КОИБ — структура, промпт, установка и перенос

Этот документ описывает VK-бота на **Bots Long Poll API**, который работает поверх
готового индекса КОИБ. В отличие от Callback API (нужен публичный HTTPS-домен),
Long Poll сам опрашивает серверы VK — серверу с ботом хватает исходящего доступа
в интернет, без белого IP и сертификата.

У вас есть два внешних API: **VK** (приём/отправка сообщений) и **GigaChat**
(генерация ответов). Эмбеддинги и поиск (FAISS + BM25) работают локально по
индексу из `./output`.

## 2. Структура бота и дерево меню

Бот переиспользует весь «тяжёлый» функционал базового `VKBotService`:
rate-limit (на пользователя и общий), дедупликацию событий, выбор и запоминание
модели, RAG-пайплайн, отправку длинных ответов по частям и прикрепление
релевантных рисунков. Сверху добавлена навигация по меню.

```
                         ┌──────────── Главное меню ────────────┐
                         │  🗳 Выбрать модель КОИБ               │
   /start, «меню»  ─────▶│  💬 Общие вопросы                    │
                         │  ❓ Частые вопросы                    │
                         │  ℹ️ Помощь                            │
                         └───────────────┬──────────────────────┘
            ┌────────────────────────────┼────────────────────────────┐
            ▼                            ▼                            ▼
   ┌─── Модель КОИБ ───┐        💬 Общие вопросы            ┌─ Частые вопросы ─┐
   │  КОИБ-2010        │   (режим «любая модель»,           │  🔌 Не включается │
   │  КОИБ-2017А       │    без жёсткого фильтра —          │  📄 Не принимает  │
   │  КОИБ-2017Б       │    ответы по всем моделям)         │  ⚙️ Замятие       │
   │  Не знаю / любая  │                                    │  … (8 вопросов)   │
   │  ⬅️ Назад          │                                    │  ⬅️ Назад          │
   └───────────────────┘                                    └───────────────────┘
```

Принципы навигации:
- **Без серверного состояния**: каждая кнопка несёт в `payload` своё действие
  (`nav` / `set_model` / `faq` / `help`). Кнопка «Назад» — это просто переход в
  нужное меню, поэтому «застрять» в подменю невозможно.
- **Кнопки — текстовые** (`type=text`): при нажатии VK присылает обычное
  `message_new` с `payload`, всё проходит через штатный разбор. Отдельный тип
  событий `message_event` не нужен.
- **Свободный текст работает всегда**: пользователь может игнорировать меню и
  просто писать вопросы. Если модель не выбрана, бот один раз спросит её
  кнопками и запомнит ответ (логика базового сервиса).

### Команды

| Команда (текст)                          | Действие                                            |
|------------------------------------------|-----------------------------------------------------|
| `/start`, `меню`, `/menu`, `начать`      | Приветствие + главное меню                          |
| `назад`, `back`                          | Вернуться в главное меню                            |
| `частые вопросы`, `/faq`                 | Раздел частых вопросов                              |
| `выбрать модель`                         | Раздел выбора модели КОИБ                           |
| `общие вопросы`                          | Режим «любая модель»                                |
| `/model`                                 | Сменить модель (inline-клавиатура базового сервиса) |
| `/help`, `помощь`                        | Справка                                             |
| `/reset`                                 | Очистить историю диалога                            |
| `/health`                                | Статус (только для `VK_ADMIN_IDS`)                  |

### Режимы «модель vs общее»

Интерфейсы различаются: у **КОИБ-2010** — физические кнопки (ДА/НЕТ), у
**КОИБ-2017А/Б** — сенсорный экран. Поэтому:
- выбор конкретной модели → ответы жёстко фильтруются по ней
  (`model_filter` + `strict_model`, флаг `MODEL_FILTER_STRICT`);
- «Общие вопросы» / «Не знаю / любая» → фильтр не жёсткий, поиск по всем
  моделям; в шапке ответа указывается, что модель не задана.

## 3. Промпт для GigaChat

Системный промпт уже есть в `src/generation.py` (`SYSTEM_PROMPT`) и работает
из коробки. Для чат-формата VK добавлена усиленная редакция в
`src/vk_prompt.py` (`VK_BOT_SYSTEM_PROMPT`): короче абзацы, явная поддержка
режима «общие вопросы», более строгий запрет копировать документацию дословно.

Что промпт гарантирует:
1. **Только КОИБ, только по индексу.** Факты берутся исключительно из
   `<retrieved_context>`; выдумывать номера пунктов, коды ошибок, контакты — нельзя.
2. **Понятный язык.** Пересказ простыми словами, нумерованные шаги-действия,
   без служебных пометок (`Фрагмент N`, `passage:`).
3. **Разделение моделей.** Действия 2010 не переносятся на 2017 и наоборот.
4. **Процессуальное напоминание.** При нештатных ситуациях добавляется блок
   «Регламентное уведомление: …» (его сохраняют валидатор и второй проход LLM).
5. **Границы.** Юридические/избирательные решения — к председателю комиссии;
   попытки подмены инструкций отклоняются.
6. **Источники.** В конце — `[Документ: имя_файла, стр. N]` по реальным страницам.

Включение усиленного промпта (без правки кода):
```
# в .env
KOIB_VK_CUSTOM_PROMPT=true
```
`run_longpoll.py` при старте подменит `src.generation.SYSTEM_PROMPT`. Если флаг
не задан — используется базовый промпт.

Поверх основной генерации работает второй проход (`ANSWER_REFINE_ENABLED=true`):
LLM переписывает черновик дружелюбным языком, сохраняя факты, ссылки на
источники и регламентный блок.

## 4. Настройка сообщества VK

1. **Токен сообщества.** Сообщество → *Управление* → *Работа с API* →
   *Ключи доступа* → создать ключ со scope **«Сообщения»** и **«Управление»**
   (manage обязателен для `groups.getLongPollServer`). Вставить в `.env`
   → `VK_ACCESS_TOKEN`.
2. **ID сообщества** (число) → `.env` → `VK_GROUP_ID`.
3. **Включить Long Poll.** Сообщество → *Управление* → *Работа с API* →
   *Long Poll API* → включить, версия API `5.131`.
4. **Типы событий.** В том же разделе включить **`Входящее сообщение`**
   (`message_new`).
5. **Сообщения сообщества.** *Управление* → *Сообщения* → включить, и в
   *Настройках для бота* разрешить добавление в беседы (если нужны беседы;
   тогда `VK_REPLY_IN_GROUP_CHATS=true`).

Для Long Poll переменные `VK_CONFIRM_CODE` и `VK_SECRET_KEY` (от Callback API)
не нужны.

## 5. Установка с нуля (Ubuntu 22.04/24.04, 2 ГБ ОЗУ)

### 5.1. Системные пакеты, git, nano, tree

```bash
sudo apt update && sudo apt upgrade -y
sudo apt install -y python3 python3-venv python3-pip git nano tree htop curl rsync unzip
python3 --version            # ожидается 3.10+
```

### 5.2. Свопфайл (страховка от OOM при загрузке модели эмбеддингов)

```bash
sudo bash deploy/setup_swap.sh   # создаёт 2 ГБ swap (скрипт из базового проекта)
free -h
```

### 5.3. Каталог проекта и виртуальное окружение

```bash
sudo mkdir -p /opt/koib-server
sudo chown "$USER":"$USER" /opt/koib-server
cd /opt/koib-server

# код проекта (через git ИЛИ перенос архивом — см. раздел 6)
git clone <URL_ВАШЕГО_РЕПО> .        # если код в git
# либо распакуйте перенесённый архив с кодом сюда

python3 -m venv venv
source venv/bin/activate
python -m pip install --upgrade pip
```

### 5.4. Зависимости (важен порядок!)

```bash
# 1) сначала CPU-сборка torch (~190 МБ вместо ~2.5 ГБ CUDA)
pip install -r requirements-torch-cpu.txt
# 2) затем серверные зависимости (faiss, sentence-transformers, aiohttp, …)
pip install -r requirements-server.txt
```
Дополнительных пакетов для Long Poll не требуется — `aiohttp` уже в
`requirements-server.txt`.

### 5.5. Конфигурация

```bash
cp .env.longpoll.example .env
nano .env
# заполните: GIGACHAT_CREDENTIALS, VK_ACCESS_TOKEN, VK_GROUP_ID
# сверьте: LOCAL_EMBEDDING_MODEL / PASSAGE_PREFIX / QUERY_PREFIX
#          ДОЛЖНЫ совпадать со значениями при индексации
```

### 5.6. Проверка структуры (tree)

```bash
tree -L 2 -I 'venv|__pycache__|.git|.hf-cache' .
```

## 6. Перенос проекта и индекса на сервер

Индекс собирается на мощной машине, на сервер переносится **готовый** `output/`.

### 6.1. На машине-индексаторе: упаковать индекс

```bash
python -m tools.index_transfer pack --output-dir ./output --out index_bundle.zip
```

### 6.2. Перенести код и индекс на сервер

```bash
# код проекта (если не через git) — пример rsync, исключая тяжёлое
rsync -avz --exclude venv --exclude output --exclude '__pycache__' \
      --exclude '.git' --exclude '.hf-cache' \
      ./  user@SERVER_IP:/opt/koib-server/

# архив с индексом
scp index_bundle.zip user@SERVER_IP:/opt/koib-server/
```

### 6.3. На сервере: распаковать и проверить индекс

```bash
cd /opt/koib-server
source venv/bin/activate
python -m tools.index_transfer unpack --archive index_bundle.zip --output-dir ./output
# распакует, перепишет пути рисунков и сверит манифест с вашим .env
python -m tools.index_transfer verify --output-dir ./output
```

## 7. Запуск

### 7.1. Вручную (проверка)

```bash
cd /opt/koib-server
source venv/bin/activate
python run_longpoll.py
# в логах: "VK Long Poll запущен | group_id=… | wait=25с"
# напишите боту в ЛС сообщества «/start» — должно прийти меню
```

или скриптом с оптимальными переменными окружения:
```bash
bash deploy/run_longpoll.sh
```

### 7.2. Как сервис (systemd, автозапуск + перезапуск)

```bash
sudo useradd -r -s /usr/sbin/nologin koib || true
sudo chown -R koib:koib /opt/koib-server

sudo cp deploy/koib-vk-longpoll.service /etc/systemd/system/
sudo systemctl daemon-reload
sudo systemctl enable --now koib-vk-longpoll
sudo systemctl status koib-vk-longpoll
journalctl -u koib-vk-longpoll -f      # живой лог
```

## 8. Частые проблемы

| Симптом                                   | Причина / решение                                                            |
|-------------------------------------------|------------------------------------------------------------------------------|
| `Нужны VK_ACCESS_TOKEN и VK_GROUP_ID`     | Не заполнен `.env`.                                                           |
| `VK API groups.getLongPollServer: …`      | У токена нет scope **manage**, либо не включён Long Poll в настройках группы. |
| Бот не отвечает на сообщения              | Не включён тип события `message_new`; проверьте, что бот может писать в ЛС.   |
| `Ошибка: GIGACHAT_CREDENTIALS не заданы`  | Пустой `GIGACHAT_CREDENTIALS` в `.env`.                                       |
| Поиск выдаёт нерелевантное / пусто        | `.env` расходится с манифестом индекса (модель/префиксы). Сверьте `verify`.   |
| Падает по памяти (OOM)                    | Включите swap (5.2), держите `USE_RERANKER=false`, `USE_HYDE=false`.          |
| Клавиатура не появляется                  | VK кеширует клавиатуру; отправьте `/start` повторно.                          |

Бот можно запускать **одновременно** с HTTP-сервером (`serve_app.py`) — это
разные процессы. На 2 ГБ ОЗУ держите оба под контролем памяти (каждый грузит
свою копию модели эмбеддингов; при нехватке памяти запускайте только бота).
